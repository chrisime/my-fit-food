# i18n — Internationalisierung

## Ziel
Alle Frontend-Strings (Fehler, Erfolge, Validierung, UI) über vue-i18n verwalten. Backend liefert nur noch Englisch + ErrorCode.

## Backend
Alle `detail`-Strings in `AppException`-Aufrufen auf Englisch normalisiert.

## Frontend

### vue-i18n Setup
- Installation mittels `pnpm add vue-i18n@9 --filter web`
- `web/src/i18n/index.ts` — `createI18n()` mit `locale` aus localStorage
- Registration in `main.ts`

### Locale Files
Flat JSON pro Sprache: `web/src/i18n/locales/{en,pt,de}.json`
Keys hierarchisch: `error.{code}`, `success.{key}`, `validation.{key}`

### useToast Integration
- `toast(key, variant)` ruft `t(key)` auf — Views übergeben Keys, keine Strings
- `toastError(err)` mapped `err.code` → `t('error.' + err.code)`
- Fallback: `error.unknown`

### LanguageSwitcher
Neue Komponente, ruft `setLocale()` auf. Persistiert in localStorage.
